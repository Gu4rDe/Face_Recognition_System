## Context

Текущий код в `ErrorMapper.kt` обрабатывает HTTP 403 как `403 -> "Доступ запрещён"`. Но сервер может возвращать более информативное сообщение в JSON:
```json
{"detail": "заблокирован"}
```
Это сообщение не парсится и теряется.

## Goals / Non-Goals

**Goals:**
- Показывать пользователю конкретное сообщение об ошибке от сервера вместо общего "Доступ запрещён"

**Non-Goals:**
- Изменять логику работы HTTP кодов, отличных от 403
- Менять API сервера

## Decisions

** Decision: Использовать существующую функцию parseDetail для 403**

Альтернативы:
1. Добавить новый парсер для 403 — избыточно
2. Использовать parseDetail как fallback — использует существующий код

Используем второй вариант: `403 -> parseDetail(e.message) ?: "Доступ запрещён"`

## Risks / Trade-offs

Минимальные риски — изменение тривиальное, затрагивает одну строку кода.
