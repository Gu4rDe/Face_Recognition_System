## ADDED Requirements

### Requirement: Username validation
Система ДОЛЖНА валидировать поле логина при регистрации и входе. Логин должен быть не короче 3 символов, содержать только латинские буквы, цифры и подчёркивания, без пробелов.

#### Scenario: Username too short
- **WHEN** пользователь вводит логин короче 3 символов и нажимает кнопку
- **THEN** отображается ошибка "Логин должен содержать минимум 3 символа"

#### Scenario: Username with invalid characters
- **WHEN** пользователь вводит логин с кириллицей или спецсимволами
- **THEN** отображается ошибка "Логин может содержать только латинские буквы, цифры и подчёркивания"

#### Scenario: Username with spaces
- **WHEN** пользователь вводит логин с пробелами
- **THEN** отображается ошибка "Логин не должен содержать пробелы"

#### Scenario: Valid username
- **WHEN** пользователь вводит корректный логин (например "admin_1")
- **THEN** ошибка не отображается

### Requirement: Email validation
Система ДОЛЖНА валидировать поле email при регистрации. Email должен содержать `@` и домен.

#### Scenario: Empty email
- **WHEN** пользователь оставляет поле email пустым
- **THEN** отображается ошибка "Введите email"

#### Scenario: Email without @
- **WHEN** пользователь вводит email без символа @
- **THEN** отображается ошибка "Введите корректный email (например, user@example.com)"

#### Scenario: Email without domain
- **WHEN** пользователь вводит email без домена (например "user@")
- **THEN** отображается ошибка "Введите корректный email (например, user@example.com)"

#### Scenario: Valid email
- **WHEN** пользователь вводит корректный email
- **THEN** ошибка не отображается

### Requirement: Password validation
Система ДОЛЖНА валидировать поле пароля при регистрации и входе. Пароль должен быть не короче 8 символов, содержать буквы, цифры и спецсимволы.

#### Scenario: Password too short
- **WHEN** пользователь вводит пароль короче 8 символов
- **THEN** отображается ошибка "Пароль должен содержать минимум 8 символов"

#### Scenario: Password without digits
- **WHEN** пользователь вводит пароль без цифр
- **THEN** отображается ошибка "Пароль должен содержать хотя бы одну цифру"

#### Scenario: Password without letters
- **WHEN** пользователь вводит пароль без букв
- **THEN** отображается ошибка "Пароль должен содержать хотя бы одну букву"

#### Scenario: Password without special characters
- **WHEN** пользователь вводит пароль без спецсимволов
- **THEN** отображается ошибка "Пароль должен содержать хотя бы один спецсимвол (!@#$%^&*)"

#### Scenario: Valid password
- **WHEN** пользователь вводит пароль "qq12345!@"
- **THEN** ошибка не отображается

### Requirement: Invite code validation
Система ДОЛЖНА проверять что код приглашения не пустой.

#### Scenario: Empty invite code
- **WHEN** пользователь оставляет поле кода приглашения пустым
- **THEN** отображается ошибка "Введите код приглашения"

### Requirement: Field hints
Система ДОЛЖНА показывать подсказки (hint) под полями ввода с требованиями к формату.

#### Scenario: Username hint
- **WHEN** поле логина пустое и не в фокусе
- **THEN** под полем отображается серый текст "Минимум 3 символа, латиница, цифры, _"

#### Scenario: Email hint
- **WHEN** поле email пустое и не в фокусе
- **THEN** под полем отображается серый текст "Например: user@example.com"

#### Scenario: Password hint
- **WHEN** поле пароля пустое и не в фокусе
- **THEN** под полем отображается серый текст "Минимум 8 символов, буквы, цифры и спецсимволы"

#### Scenario: Invite code hint
- **WHEN** поле кода приглашения пустое и не в фокусе
- **THEN** под полем отображается серый текст "Получите код от администратора"

### Requirement: Server error instructions
Система ДОЛЖНА показывать инструкции после серверных ошибок с рекомендациями.

#### Scenario: HTTP 401 login error
- **WHEN** сервер возвращает 401 при входе
- **THEN** отображается "Неверный логин или пароль" и инструкция "Проверьте правильность ввода и попробуйте снова"

#### Scenario: HTTP 403 registration error
- **WHEN** сервер возвращает 403 при регистрации
- **THEN** отображается detail из ответа и инструкция "Убедитесь, что ваш код приглашения действителен"

#### Scenario: HTTP 409 conflict error
- **WHEN** сервер возвращает 409
- **THEN** отображается "Пользователь уже существует" и инструкция "Попробуйте другой логин или email"

#### Scenario: HTTP 400 or 422 error
- **WHEN** сервер возвращает 400 или 422
- **THEN** отображается detail из ответа и инструкция "Проверьте правильность заполнения всех полей"
