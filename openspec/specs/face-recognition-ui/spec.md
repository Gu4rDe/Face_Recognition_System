## ADDED Requirements

### Requirement: Загрузка фото для распознавания

FaceRecognitionContent SHALL предоставлять два режима загрузки фотографии: выбор файла через JFileChooser и захват с веб-камеры через PhotoCaptureDialog. Оба режима устанавливают источник изображения (ByteArray) для последующей отправки на распознавание через ServiceLocator.faceRecognitionRepository.recognizeFace(imageBytes).

#### Scenario: Успешная загрузка файла и распознавание

- **WHEN** пользователь выбирает файл фотографии через кнопку «Выбрать фото» и нажимает «Распознать»
- **THEN** фото отправляется на сервер, возвращаются результаты распознавания

#### Scenario: Успешный захват с камеры и распознавание

- **WHEN** пользователь нажимает кнопку «Снять с камеры», захватывает фото в PhotoCaptureDialog и нажимает «Распознать»
- **THEN** захваченное фото используется как источник, отправляется на сервер, возвращаются результаты распознавания

#### Scenario: Отмена захвата с камеры

- **WHEN** пользователь нажимает «Отмена» в PhotoCaptureDialog
- **THEN** диалог закрывается, изображение не изменяется, камера освобождается

#### Scenario: Ошибка распознавания

- **WHEN** распознавание завершается ошибкой
- **THEN** отображается сообщение об ошибке на русском языке (через ErrorMapper)

#### Scenario: Изображение не выбрано

- **WHEN** пользователь нажимает «Распознать» без выбора файла или захвата с камеры
- **THEN** кнопка «Распознать» неактивна

#### Scenario: Переключение источника изображения

- **WHEN** пользователь выбирает файл после захвата с камеры (или наоборот)
- **THEN** предыдущий источник заменяется новым, результат распознавания сбрасывается

### Requirement: Отображение результатов распознавания

FaceRecognitionContent SHALL отображать результаты распознавания: количество обнаруженных лиц (faces_detected) и список совпадений для каждого лица с username, similarity и bounding box. Для каждого лица с совпадениями отображается подпись «{username} ({similarity%})».

#### Scenario: Лица обнаружены с совпадениями

- **WHEN** распознавание успешно и найдены лица с совпадениями
- **THEN** отображается список результатов: для каждого лица — bounding box с подписью лучшего совпадения

#### Scenario: Лица обнаружены без совпадений

- **WHEN** распознавание успешно, лица найдены, но совпадений нет
- **THEN** отображаются bounding box без подписей совпадений

#### Scenario: Лица не обнаружены

- **WHEN** распознавание успешно, но лица не найдены
- **THEN** отображается сообщение «Лица не обнаружены»

### Requirement: Визуализация bounding box на изображении

FaceRecognitionContent SHALL отображать прямоугольники bounding box поверх изображения предпросмотра для каждого обнаруженного лица. Прямоугольники SHALL быть цветово кодированы по качеству лучшего совпадения: зелёный (>порог совпадения), жёлтый/оранжевый (75%–порог), красный (<75% от порога). Для лиц без совпадений используется серый цвет. Подпись над прямоугольником SHALL содержать имя лучшего совпадения и процент сходства. Порог совпадения определяется настройкой matchThreshold из SettingsState.

#### Scenario: Отображение bounding box с высоким сходством

- **WHEN** результат распознавания содержит лицо с совпадением similarity > matchThreshold
- **THEN** прямоугольник bounding box отображается зелёным цветом с подписью «{username} ({similarity%})»

#### Scenario: Отображение bounding box со средним сходством

- **WHEN** результат распознавания содержит лицо с лучшим совпадением similarity от (matchThreshold × 0.75) до matchThreshold
- **THEN** прямоугольник bounding box отображается жёлто-оранжевым цветом с подписью «{username} ({similarity%})»

#### Scenario: Отображение bounding box с низким сходством

- **WHEN** результат распознавания содержит лицо с лучшим совпадением similarity < (matchThreshold × 0.75)
- **THEN** прямоугольник bounding box отображается красным цветом с подписью «{username} ({similarity%})»

#### Scenario: Отображение bounding box без совпадений

- **WHEN** результат распознавания содержит лицо без совпадений (matches пуст)
- **THEN** прямоугольник bounding box отображается серым цветом без подписи совпадения

#### Scenario: Масштабирование координат bounding box

- **WHEN** размер отображаемого изображения отличается от оригинального
- **THEN** координаты bounding box масштабируются пропорционально с учётом ContentScale.Fit (letterboxing)

### Requirement: Захват фото с веб-камеры

FaceRecognitionContent SHALL предоставлять кнопку «Снять с камеры» для захвата фото через существующий PhotoCaptureDialog. Захваченное фото SHALL использоваться как источник для распознавания.

#### Scenario: Кнопка захвата с камеры доступна

- **WHEN** экран распознавания лиц отображается
- **THEN** кнопка «Снять с камеры» отображается рядом с кнопкой «Выбрать фото»

#### Scenario: Захват с камеры успешен

- **WHEN** пользователь нажимает «Снять с камеры», захватывает фото и подтверждает в PhotoCaptureDialog
- **THEN** захваченное фото отображается в предпросмотре и используется как источник для распознавания

### Requirement: Данные сотрудников в результатах распознавания

FaceRecognitionContent SHALL обогащать результаты распознавания данными сотрудников: для каждого совпадения отображается полная информация (должность, отдел, email, телефон, местоположение, статус активности, доступ).

#### Scenario: Обогащение данных сотрудника

- **WHEN** распознавание успешно и совпадения найдены
- **THEN** для каждого совпадения загружаются данные сотрудника через searchEmployees(username) и отображается расширенная информация

#### Scenario: Данные сотрудника недоступны

- **WHEN** данные сотрудника для совпадения не найдены
- **THEN** отображается только username и сходство из результата распознавания
